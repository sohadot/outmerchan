The Outmerchant Protocol

The Outmerchant Protocol defines how Merchant Sovereignty is named, measured, explained, and protected across the asset.

The lexicon, the score, the reference pages, and the ranks are not separate features. They are four functions of one system, and this page states how they bind together. A definition that is not measured is rhetoric; a measurement that is not defined is arbitrary; an explanation that can drift from both is decoration. The protocol exists so that none of these failures can occur here.

1. Named — The Lexicon

The Naming Law

Every concept the standard uses is defined once, canonically, in the Outmerchant Lexicon. No term enters without a definition, a function inside the system, a relationship to other terms, and a place in the measurement. Definitions are published verbatim from the governed structured record — the lexicon a machine reads and the lexicon a human reads are the same lexicon.

2. Measured — The Score

The Measurement Law

The Merchant Sovereignty Score measures the condition the lexicon names: 0–100, across eight governed dimensions of control, classified into four tiers. The model — dimensions, weights, questions, boundaries — lives in public structured files and changes only through a recorded decision. Every surface that computes or displays a score reads from that model; none may carry its own.

3. Explained — The Reference Standard

The Explanation Law

Every dimension and every tier of the score has a reference page — from Platform Dependency through Governance Independence, and from Captured to the terminal rank. Reference pages are generated from the governed model itself, never written separately from it: the explanation cannot drift from the thing it explains.

4. Protected — The Governance

The Protection Law

An automated quality gate validates the whole system on every change: model integrity, definition fidelity, link topology, indexability, and the inviolable elements. Chief among the protected elements is the terminal rank: the highest tier of the score permanently carries the name Outmerchant (decisions DEC-001, DEC-006, DEC-016). Every significant decision about the system is recorded in a public decision log — the asset does not merely have pages; it has reasons.

Every layer serves the merchant.
No layer owns the merchant.

The protocol's posture is deliberately bounded: not anti-platform, not anti-state, not anti-bank — anti-single-point-of-control. The standard does not campaign against infrastructure; it measures the merchant's independence from any single piece of it, and names the condition in which that independence is complete.
